# Research-grade 报告质量标准

## 1. 报告完整性

一份可直接投研阅读的细分方向报告至少包含：

1. 结论摘要：主线地位、兑现阶段、估值水位、资金拥挤度、优先级。
2. 产业逻辑：需求来源、价值量变化、技术路线、传导路径。
3. 产业链位置：上游、中游、下游、交叉主线。
4. 代表公司对比：收入暴露、客户/订单、产能/产品、财务、估值、风险。
5. 业绩与估值：历史收入利润、毛利率、PE TTM、PS TTM、2026E/2027E、PEG或缺口说明。
6. 行情与资金：近1/3/6月涨幅、20日成交额、相对指数强弱、拥挤度。
7. 催化剂：产业、政策、公司、财报四类。
8. 风险与证伪：估值、业绩、技术路线、竞争、客户导入。
9. 数据来源与缺口：来源索引、缺失清单、冲突清单。

## 2. 来源要求

关键判断必须至少有一个可索引来源：

- 本地缓存：`investment_system/data/raw/...`
- 本地 evidence：`investment_system/research/evidence/...`
- 网页 URL：公司公告、交易所、互动平台、政府文件、研报页面、数据库页面。

来源行必须包含：

```text
source_id, source_type, source_name, source_date, source_url,
related_company, quote_or_excerpt, data_fields_supported, confidence_level
```

不允许出现只有“网上资料”“研报显示”“市场认为”但没有 URL 或本地路径的来源描述。

## 3. 正文禁用占位

research-grade 正文中禁止出现：

- `待核实`
- `调试级`
- `待精确URL`
- `后续补`
- `缺失；待`
- `研报终端核实项`

这些内容只能出现在 `数据缺口` 或 `缺失数据清单.md` 中。

## 4. 联网补采质量

当接口无法获得数据时，可以联网搜索，但要满足：

- 优先一级来源：公司报告、公告、交易所、政府文件。
- 二级来源可用于预测和行业判断：券商研报、终端一致预期页面、权威财经数据库。
- 媒体和社区只能作为低置信线索。
- 每条联网事实都要进入 `source_rows`，并在报告正文引用其 `source_id` 或来源名称。

## 5. 验收分级

`pipeline-grade` 验收关注文件、字段、schema 和基础缺失。

`research-grade` 验收额外关注：

- 报告章节完整。
- 公司覆盖完整。
- 正文没有调试占位。
- 来源 URL 或本地路径可索引。
- 关键结论不是无来源断言。
- 缺失项集中披露，不污染正文。
