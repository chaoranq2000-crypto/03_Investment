# 日常工作流

## 盘前

1. 查看持仓和观察池。
2. 检查昨晚数据快照是否完整。
3. 更新今日关注事项：财报、公告、行业新闻、技术位。
4. 形成今日实时操作计划：买入候选、卖出条件、触发价格、仓位上限。

## 盘中

1. 只对重点标的查询实时行情。
2. 不批量硬拉 AKShare。
3. 记录异动：放量、涨跌停、资金流、板块联动。
4. 根据实时操作决策卡执行或取消操作。
5. 盘中判断只服务交易决策，不替代完整研究。

## 收盘后

1. 批量刷新日线、行情快照、资金流。
2. 更新本地缓存和指标表。
3. 跑股票池筛选和评分。
4. 对新入选标的生成研究任务。
5. 更新实时操作决策卡状态。
6. 更新组合复盘。

## 个股研究流程

```text
基础资料 -> 财务质量 -> 业务拆分 -> 行业空间 -> 竞争格局 -> 催化剂 -> 估值 -> 风险 -> 结论 -> 实时操作计划
```

输出建议：

- 一页摘要
- 核心假设表
- 财务指标表
- 催化剂时间线
- 风险清单
- 跟踪计划

## 实时操作决策流程

每个实际操作都应该先形成一张决策卡：

```text
研究结论 -> 盘前计划 -> 实时触发 -> 执行动作 -> 状态跟踪 -> 收盘复盘
```

决策类型：

- 买入：价格和研究条件同时满足，允许开仓。
- 加仓：原假设继续被验证，且风险收益比仍合适。
- 减仓：上涨兑现、风险收益比下降或实时信号转弱。
- 清仓：核心逻辑被证伪，或触发风险条件。
- 持有：研究逻辑未变，实时波动不触发操作。
- 观察：研究有吸引力，但价格、催化、仓位或证据不足。

实时触发条件可以包括：

- 价格到达计划区间。
- 突破或跌破关键均线/平台。
- 放量、缩量、涨跌停、异动拉升。
- 主力资金流或板块联动变化。
- 公告、财报、政策、行业事件落地。
- 组合仓位或风险暴露达到上限。

操作原则：

- 没有盘前计划，不做临时冲动操作。
- 没有触发条件，不执行操作。
- 没有失效条件，不允许重仓。
- 每次执行后必须留下复盘入口。

## 报告命名

建议格式：

```text
reports/{股票代码}_{公司名}_{主题}_{YYYYMMDD}.md
```

示例：

```text
reports/600141_兴发集团_磷化铟收入测算_20260619.md
```

## 数据文件命名

建议格式：

```text
data/raw/{source}/{dataset}/{date}.json
data/cache/{source}/{dataset}/{date}.parquet
data/processed/{dataset}/{date}.parquet
```

这样可以按天增量写入，不需要重写历史文件。
