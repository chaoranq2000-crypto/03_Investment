# 项目日志总索引

Project: `tech_ai_semiconductor`
Last updated: 2026-06-29
Purpose: lightweight catalog for choosing project logs without reading log bodies.

## 默认规则

- 没有用户明确指令时，默认不读取任何项目日志。
- 没有用户明确指令时，默认不创建、更新或追加任何项目日志。
- 需要查找可选日志时，只读取本索引；不要为了找日志而读取日志正文。
- 只有用户明确说 `读日志 <中文日志名>` 时，才读取被选中的那一个日志或文档。
- 只有用户明确说 `写日志`、`写交接`、`写决策` 或要求归档/记录时，才写入日志。
- 一个对话默认只维护一个主日志；若已存在本对话主日志，追加到同一文件。

## 指令到动作

| 用户指令 | 允许读取 | 允许写入 |
|---|---|---|
| 未提日志 | 不读取日志或本索引 | 不写日志 |
| `不读日志` | 不读取日志或本索引 | 仍需另行明确是否写 |
| `不写日志` | 仅按用户明确选择读取 | 不写日志 |
| `只列日志` | 只读取本索引 | 不写日志 |
| `读日志菜单` | 只读取本索引 | 不写日志 |
| `读日志 <中文日志名>` | 先用本索引定位，再只读被选中的一项 | 不写日志，除非另有 `写日志` |
| `写日志` | 按任务需要读取同一主日志 | 写入一个主会话日志 |
| `写交接` | 按任务需要读取同一主日志 | 写入交接记录或追加主日志 |
| `写决策` | 按任务需要读取同一主日志 | 写入决策记录或追加主日志 |

- 新增、删除或合并日志后，优先更新本索引。
