# Codex 调研说明手册：A股科技前两主线

生成日期：2026-06-19  
适用范围：AI算力硬件、半导体国产替代两条主线。  
目标：让 Codex 流程化搜集信息，形成可横向比较的数据底稿，再交由 ChatGPT/人工进行投资判断。

---

## 0. 总原则

1. **Codex 不直接给买卖建议**，只负责搜集、核实、结构化和初步打分。
2. 每一条关键事实必须保留：**来源名称、发布日期、链接、原文摘录**。
3. 无法确认的数据必须写“缺失”或“无法确认”，不得猜测。
4. 代表公司只是初始清单，不代表推荐；必须核实是否真的有收入暴露。
5. 每个细分方向都要回答四个核心问题：
   - 这个方向是不是当前科技主线？
   - 业绩是否已经兑现或即将兑现？
   - 当前估值是否已经透支？
   - 资金是否认可且拥挤度是否过高？

## 3. 数据来源优先级

优先级从高到低：

1. 公司年报、半年报、季报、公告。
2. 投资者关系活动记录。
3. 交易所互动平台回复：互动易、上证e互动、北交所问答。
4. 机构一致预期、券商研报摘要、Wind/同花顺/iFind/东方财富 Choice 等终端数据。
5. 权威媒体和产业媒体。
6. 普通财经号、社区观点只能作为线索，不可直接作为事实依据。

每条数据至少包含：

```text
source_type, source_name, source_date, source_url, quote_or_excerpt, confidence_level
```

## 6. 业绩兑现阶段评分

| 分数 | 定义 |
|---:|---|
| 5 | 已经大规模贡献收入和利润，财报中可验证 |
| 4 | 已有订单、客户或定点，收入开始放量 |
| 3 | 已有验证、小批量交付或明确产能建设 |
| 2 | 样品、送样、技术储备阶段 |
| 1 | 主要是概念、政策或市场传闻 |
| 0 | 无法确认与该主线相关 |

Codex 必须用证据支撑评分，不能只写主观判断。

## 7. 综合评分框架

建议先用粗评分，后续再由 ChatGPT/人工修正：

| 维度 | 权重 | 评分方向 |
|---|---:|---|
| 产业景气度 | 20% | 越高越好 |
| 业绩兑现概率 | 25% | 越高越好 |
| 上涨空间 | 20% | 越高越好 |
| 泡沫风险 | 15% | 风险越低分越高 |
| 资金认可度 | 10% | 越高越好 |
| 催化强度 | 10% | 越高越好 |

综合评分公式：

```text
score = 景气度*20 + 兑现概率*25 + 上涨空间*20 + 泡沫安全分*15 + 资金认可度*10 + 催化强度*10
```

其中每个子项用 0-5 分，最后换算到 0-100 分。

## 8. 泡沫风险判断

泡沫风险不是简单看涨幅，而是综合：

1. 近3月/6月涨幅是否显著高于业绩上修幅度。
2. PE/PS/PEG 是否明显高于历史分位和同行。
3. 交易是否拥挤：成交额、换手率、龙虎榜、融资余额。
4. 是否只有远期故事，没有近两年利润兑现。
5. 是否出现“利好密集释放 + 股价加速 + 放量滞涨”。

泡沫安全分定义：

| 分数 | 含义 |
|---:|---|
| 5 | 估值合理，涨幅不大，业绩能支撑 |
| 4 | 略有溢价，但业绩兑现较强 |
| 3 | 有一定拥挤，需要等回调或只看龙头 |
| 2 | 明显高估，短期情绪占比高 |
| 1 | 高泡沫，已按远期乐观情景定价 |
| 0 | 数据缺失，无法判断 |

## 9. 缺失数据处理

如果数据缺失，按以下格式记录：

```markdown
| 细分方向 | 公司 | 缺失字段 | 已查来源 | 下一步建议 |
|---|---|---|---|---|
```

不要为了填满表格而编造数据。

---

## 10. 冲突数据处理

如果不同来源数据冲突，按以下优先级处理：

1. 公司公告 > 定期报告 > 投资者关系记录 > 交易所互动 > 数据终端 > 媒体报道 > 社区观点。
2. 保留所有冲突来源，并在“冲突数据清单.md”中记录。
3. 不强行选一个结论，除非高优先级来源明确覆盖低优先级来源。
