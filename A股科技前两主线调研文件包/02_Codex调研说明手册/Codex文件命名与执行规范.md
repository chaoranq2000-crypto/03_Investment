# Codex 文件命名与执行规范

## 1. 命名规范

- 细分方向文件名：`序号_细分方向.md`
- 公司表：统一写入 `代表公司财务估值总表.csv`
- 来源表：统一写入 `数据来源索引.csv`
- 缺失数据：统一写入 `缺失数据清单.md`
- 冲突数据：统一写入 `冲突数据清单.md`

## 2. 禁止事项

- 禁止将没有来源的数据写成事实。
- 禁止把“有概念”写成“有业绩”。
- 禁止用单一研报观点替代公司公告或财报数据。
- 禁止遗漏来源日期。
- 禁止把子公司、参股公司、客户关系未经核实就当作确定性收入。

## 3. 必须标记的风险

- 收入占比无法确认。
- 客户或订单仅为市场传闻。
- 估值已经显著高于同行或历史分位。
- 近3月涨幅远高于业绩上修。
- 产品仍处于送样、验证、小试阶段。
- 政策主题强但商业化时间不明。

## 4. 输出检查清单

每完成一个细分方向，检查：

- 是否有至少3个代表公司？
- 是否有行情、估值、业绩、资金、催化、风险六类信息？
- 是否标明每项数据来源？
- 是否区分已兑现与未兑现？
- 是否记录缺失数据？
- 是否能够和其他细分方向横向比较？
